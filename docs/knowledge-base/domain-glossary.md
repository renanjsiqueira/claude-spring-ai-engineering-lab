# Domain Glossary

- Backlog item: a unit of work classified as FEATURE, BUG, REFACTOR, ARCHITECTURE or QUESTION, with a
  title, user story, acceptance criteria and technical tasks.
- Project: a product context identified by a human-readable code such as `brabrix-dev`.
- Acceptance criterion: a specific, testable pass/fail condition for a backlog item, ideally written
  in Given/When/Then form.
- Technical task: a concrete implementation step needed to complete a backlog item.
- Priority: the urgency of a backlog item, one of LOW, MEDIUM, HIGH or CRITICAL.
- Complexity: the estimated implementation effort of a task, one of LOW, MEDIUM or HIGH.
